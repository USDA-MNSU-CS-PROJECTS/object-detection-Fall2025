# Legacy Image Converters

**⚠️ Note: This directory contains legacy conversion code.**

This directory contains older scripts used for converting `.nd2` files to `.tiff` format.

## Overview of the Legacy Approach

The conversion process in this directory follows this workflow:
1.  **Input**: Takes an `.nd2` file.
2.  **Normalization**: Normalizes pixel values to the 0-255 range.
3.  **8-bit Conversion**: Converts the data to 8-bit unsigned integers.
4.  **Output**: Saves the result as a `.tiff` file.

## Why this approach is deprecated

This approach was used as an intermediate solution because we initially lacked a robust method for direct conversion that preserved color fidelity without data loss. The normalization step to 8-bit could potentially result in loss of dynamic range compared to the original raw data.

## Current Recommended Approach

For the most up-to-date and accurate conversion, please refer to the new converter implementation located at:

`src/app/api/converter.py`

The new converter handles channel orientation and color preservation more effectively and supports direct conversion to PNG for web viewing.

## File Descriptions

*   **`nd2_converter.py`**:
    *   **Function**: `convert_nd2_to_tiff(input_path, output_dir)`
    *   **Description**: Reads an `.nd2` file, normalizes it to 8-bit, handles basic channel ordering (BGR to RGB), and saves it as a `.tiff` file. This was the first step in the legacy pipeline.

*   **`png_converter.py`**:
    *   **Function**: `convert_tiff_to_png(input_path, output_dir)`
    *   **Description**: Takes the intermediate `.tiff` file generated by `nd2_converter.py` and converts it to a `.png` file. This was the second step in the legacy pipeline.

*   **`__init__.py`**:
    *   **Description**: Exposes the conversion functions. **Note**: This file may contain outdated imports (referencing `tiff_converter` which appears to be missing or renamed).
